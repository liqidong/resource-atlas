# Document System Architecture

Use this when architecture knowledge needs to be packaged for README files, engineering design docs, onboarding, reviews, or stakeholder updates.

## Current Resources

- [Architecture Diagram Generator](../resources/architecture-diagram-generator.md): a low-friction way to make a shareable diagram and summary cards from a concise component-and-flow description.

## Documentation Axes

- Component boundaries and ownership
- Data flow, protocols, ports, and trust boundaries
- Cloud regions, security groups, queues, databases, and external services
- What should be visible to non-engineering readers
- Whether the generated diagram needs source-controlled HTML, exported PNG/PDF, or both
- How the artifact will be refreshed when the system changes

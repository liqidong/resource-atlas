# Taxonomy Policy

Keep taxonomy light until repeated use proves a term matters.

## Tags

- Check `data/tags.yaml` before adding a new tag.
- Prefer aliases over near-duplicate tags.
- Tags describe what the resource is about, not every topic mentioned.

## Use Cases

Use cases should be action phrases:

- `find-practical-ai-prompts`
- `study-prompt-library-maintenance`
- `build-prompt-resource-library`

Create a use case when it helps answer a future user question. Do not create one just because a resource has a topic.

## Concepts

Delay concept pages until a concept appears in at least 3 resources or becomes a clear future query anchor.
